# 全プロジェクト共通ルール

## 言語

- 応答・説明・コミットメッセージ・PR本文は日本語で書く（コード・技術用語は原文のまま）

## Git

- main / master / develop への直接コミットは禁止。作業ブランチ（feature/xxx 等）を作成してから作業する
- コミットメッセージは Conventional Commits 形式（type は英語、説明は日本語）
- commit / push / PR作成はユーザーの指示があったときのみ実行する

## Claudeが作成するmdファイル

- 調査結果・計画・レポート等のmdファイルは、プロジェクトルートの `.ai-notes/` に格納する（存在しなければ作成する）
- 例外: README・仕様書など、リポジトリのドキュメントとして明示的に指示されたもの
- 作成後は相対パスを報告する

## 品質

- 完了報告の前に、プロジェクトにある検証手段（型チェック・lint・テスト）を実行し、結果を添える
- コードは読んでから指摘・修正する。推測で書かない
