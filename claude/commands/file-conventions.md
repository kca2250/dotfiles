# ファイル共通ルール

他の skill が参照する共通のファイル配置・命名・出力ルール。

---

## 1. ファイル作成時のルール

### 配置
- 新規ファイルは既存のディレクトリ構造・命名規則に従う
- 不明な場合は既存の同種ファイルを確認してから配置を決める
- ユーザーに配置先を確認してから作成する

### 命名
- プロジェクトの既存の命名規則に従う（kebab-case, PascalCase 等）
- 既存パターンと異なる命名を避ける

---

## 2. ファイルパスの報告

ファイルを作成・変更した場合は、必ず以下を報告する:

- **相対パス** で提示（絶対パスではない）
- 複数ファイルの場合は一覧形式で提示

```
### 作成・変更ファイル
- src/components/ui/Button/index.tsx（新規作成）
- src/components/ui/Button/types.ts（新規作成）
- src/pages/Home/index.tsx（変更）
```

---

## 3. mdファイル出力時のルール

調査結果・計画等をmdファイルに出力する場合:

- ファイル名は内容を端的に表す英語名にする
- 作成後に **相対パスを必ず報告** する
- 既存の同種ドキュメントがある場合は統合を検討する（無駄に増やさない）

---

## 4. ディレクトリ構成の確認

新しいディレクトリ構造に不安がある場合:

1. 既存の類似ディレクトリを `Glob` で確認
2. そのパターンに合わせて配置
3. 判断に迷う場合はユーザーに確認
