現在のブランチの変更内容を確認し、プルリクエストを作成してください。

## 手順

1. `git log develop..HEAD --oneline` または `git log origin/develop..HEAD --oneline` でコミット一覧を確認
2. 変更内容に基づいてPRタイトルと本文を生成
3. ユーザーに確認後、`gh pr create` を実行
4. 作成後、`gh browse` でブラウザで開く

## PR作成コマンド
```bash
gh pr create --base develop --draft --assignee @me --title "タイトル" --body "本文"
```

## タイトル（日本語）

- 簡潔に変更内容を説明（50文字以内目安）
- 例: 「ログイン機能の追加」「ユーザー一覧のページネーション対応」

## 本文テンプレート（日本語）
```markdown
## 概要

このPRで行った変更の概要を記載

## 変更内容

- 変更点1
- 変更点2
- 変更点3

## 関連Issue

- Close #123（あれば）

## 確認事項

- [ ] 動作確認済み
- [ ] テスト追加/更新済み（必要な場合）
```

## ヒントが与えられた場合

$ARGUMENTS

ヒントがあれば、それを参考にしてタイトル・本文を生成する。

## 出力例

タイトル:
```
Google OAuth ログイン機能の追加
```

本文:
```markdown
## 概要

Google アカウントを使用したログイン機能を実装しました。

## 変更内容

- Google OAuth 2.0 認証フローの実装
- ログインボタンコンポーネントの追加
- 認証状態の管理をContextで実装

## 確認事項

- [ ] 動作確認済み
- [ ] テスト追加/更新済み
```

## 注意

- プッシュされていない場合は、先にプッシュするか確認
- developブランチにいる場合は、PRを作成できない旨を伝える
- ベースブランチはデフォルトで `develop` を使用
