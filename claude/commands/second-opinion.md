# セカンドオピニオン

指定された対象について、独立したサブエージェントに意見を求め、自身の見解と比較する。

対象: $ARGUMENTS

---

## 手順

### Step 1: 対象の把握

- 対象ファイル or 質問内容を確認する
- ファイルの場合は内容を読んで把握する

### Step 2: 自身の見解をまとめる

- 対象について自分なりの分析・レビュー結果を整理する
- まだユーザーには提示しない

### Step 3: サブエージェントに独立レビューを依頼する

Agent ツールでサブエージェントを起動し、独立した見解を求める。

**独立性の確保（必須）:**

- 自身の見解・結論・仮説をプロンプトに含めない（誘導・バイアスの防止）
- 対象・観点・出力形式だけを渡し、ゼロから分析させる

プロンプトテンプレート:

```
以下の対象について、独立した立場で分析・レビューしてください。

- 対象: <ファイルパス or 質問内容>
- 観点: <正確性 / 設計 / セキュリティ など、対象に応じて指定>
- 出力: 見解を箇条書きで。各項目に根拠（file:line や理由）を必ず添えること。
```

### Step 4: 比較レポート

両方の見解を以下の形式でユーザーに提示する:

```markdown
## セカンドオピニオン結果

### Claude（自身の見解）
- ...

### サブエージェント（独立レビュー）の見解
- ...

### 共通点
- ...

### 相違点
| 観点 | 自身 | サブエージェント |
|------|------|-----------------|
| ... | ... | ... |

### 総合判断
- ...
```

---

## スコープ制御

### やること
- 対象への分析・レビュー
- サブエージェントへの独立した問い合わせと結果の比較
- 相違点の明確な提示

### やらないこと
- コードの修正（比較・報告のみ）
- どちらが正しいかの断定（判断はユーザーに委ねる）
- 自身の見解をサブエージェントに事前に伝えること
