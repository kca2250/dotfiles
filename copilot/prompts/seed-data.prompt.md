# Seedデータ作成

指定された内容に基づいてseedデータを作成・更新する。

対象: $ARGUMENTS

---

## 1. 前提確認（着手前に必ず行う）

- [ ] スキーマファイル（Prisma / Drizzle / SQL等）を読了したか
- [ ] 既存のseedファイルがあれば読了し、パターンを把握したか
- [ ] 対象テーブルのリレーション（外部キー・依存関係）を把握したか
- [ ] seedの実行方法（コマンド・エントリポイント）を確認したか

---

## 2. seedデータの設計

### Step 1: 対象テーブルの特定

- seedデータを作成するテーブルを列挙する
- リレーションの依存順序を整理する（親テーブル → 子テーブル）
- ユーザーにデータ量の方針を確認する（最小限 / 開発に十分な量）

### Step 2: データ方針の確認

以下をユーザーと合意する:

| 項目 | 選択肢 |
|------|--------|
| 冪等性 | upsert / truncate+insert / 初回のみ |
| データ量 | 最小限（1-2件） / 開発用（5-10件） / 大量（性能テスト用） |
| データ内容 | 固定値 / faker等によるランダム生成 |

---

## 3. 実装手順

### Step 1: seedファイルの作成・更新

1. 既存のseedファイルがあればそのパターンに従う
2. なければプロジェクトの規約に沿った場所に新規作成する
3. リレーション順序（親 → 子）でデータを投入する

### Step 2: 冪等性の確保

- 繰り返し実行してもエラーにならないことを保証する
- upsert または 既存データの確認後に投入する方式を採用する

### Step 3: データの質

- 開発時に意味のあるデータにする（例: ユーザー名が "test1" ではなく現実的な値）
- 必須フィールドは全て埋める
- enum / ステータス系は複数パターンを含める
- 日付フィールドは過去・現在・未来を混ぜて境界値を含める

---

## 4. 実行方法の提示

seedデータの投入方法をユーザーに提示する。プロジェクトの構成を確認し、該当する実行コマンドを案内すること。

例:
- **Prisma**: `npx prisma db seed`
- **Drizzle**: `npx tsx src/db/seed.ts`
- **TypeORM**: `npx ts-node src/seeds/run.ts`
- **Rails**: `rails db:seed`
- **Django**: `python manage.py loaddata seed.json`
- **カスタムスクリプト**: `package.json` の `scripts` に定義されたコマンド（例: `npm run seed`）

プロジェクト固有のコマンドが存在する場合はそちらを優先して案内する。

---

## 5. 検証

- [ ] seedコマンドを実行してエラーが出ないこと
- [ ] 型エラー・制約違反（NOT NULL、UNIQUE、FK）がないこと
- [ ] 2回連続で実行してもエラーにならないこと（冪等性）
- [ ] 投入されたデータをアプリで表示して問題ないこと（確認可能な場合）

---

## 6. スコープ制御

### やること
- seedデータの作成・更新
- 既存seedファイルとの整合性確保
- リレーション順序の考慮

### やらないこと
- commit / push の自動実行
- 本番環境でのseed実行
- スキーマ変更（必要な場合は `/db-migrate` を案内）
- seedに関連しないコードの修正
